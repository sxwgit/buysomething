# v1 建议

1. 使用场景，他是一个给三层部门用的小系统，平常大家用的比较少，只要特定场景才会用到，所以操作要简单，覆盖我的功能即可，但是数据透视要清晰，方便人看

2. 部门大约有300人，尽管不是所有人同时，但是希望你的数据处理做优化，简单来说，就是大家上这个网站看数据不会卡，同时如果资产管理员刷新数据，可以实时同步数据，这个需要你做好设计。

3. 资产管理员都有哪些数据：
3.1 基于表格的采购数据，需要支持表格导出，也支持通过表格导入数据，请注意，如果用表格导入，目前的建议是新建，不考虑覆盖已有的数据
3.2 每次的审批邮件，将邮件可以视为一个审批记录，每月有一个保存归档即可
3.3. 上面的数据是基于管理员的添加能力，管理员我是根管理员，我可以管理其他管理员，但是所有管理员的权限（除了对管理员的管理外），都是一样的

4. 数据的代码应该清晰，基于我的现有功能和上面的描述思考下

5. 补充点，基于上面的考虑，我对安全的建议不是特别关注（除了非常影响的外）
