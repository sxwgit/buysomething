# 采购管理系统 — 开发文档

## 1. 项目概述

### 1.1 背景

目前采购管理全部依赖 Excel 手工记录，存在数据分散、协作困难、难以统计等问题。需要开发一个轻量级 Web 系统，实现采购数据的在线管理、审批附件归档、以及多维度的透视报表。

### 1.2 目标

- 在线管理采购申请数据（类似 Excel 的录入体验）
- 支持按部门、按月归档审批邮件附件
- 提供多维度的采购透视报表和图表
- 支持历史 Excel 数据导入
- 部署在内网服务器，部门内多人通过浏览器访问

### 1.3 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Python 3.10+ / Flask | 轻量 Web 框架，适合小型内部系统 |
| 数据库 | SQLite | 零配置，单文件部署，适合内网小团队 |
| ORM | Flask-SQLAlchemy | 简化数据库操作 |
| 前端 | Jinja2 模板 + Bootstrap 5 | 简洁美观，无需前后端分离 |
| 表格组件 | DataTables | 提供类 Excel 的分页、搜索、排序功能 |
| 图表 | ECharts（通过 CDN） | 饼图、柱状图等可视化 |
| Excel 处理 | openpyxl | 历史数据导入、透视结果导出 |
| 文件上传 | Flask-Upload / 原生处理 | 附件上传与存储 |

## 2. 功能需求

### 2.1 采购申请管理

#### 数据字段

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| 采购年份 | 下拉框 | 从 2022 开始到当前年份 | 必填 |
| 采购月份 | 下拉框 | 1-12 月 | 必填 |
| 资产最小分类 | 下拉框 | 预置分类数据，后期可修改 | 必填 |
| 物品名称 | 文本输入 | | 必填 |
| 生产厂家 | 文本输入 | | 非必填 |
| 物品型号 | 文本输入 | | 非必填 |
| 预算数量 | 数字输入 | 正整数 | 必填 |
| 预计单价 | 数字输入 | 保留两位小数 | 必填 |
| 总价 | 自动计算 | 预算数量 × 预计单价，页面实时计算 | 自动 |
| 需求部门 | 下拉框 | 预置部门数据，后期可修改 | 必填 |
| 需求人姓名 | 文本输入 | | 必填 |
| 需求人工号 | 文本输入 | | 必填 |
| 资产编码 | 文本输入 | 申请时即填写 | 非必填 |
| 申请原因 | 文本域 | | 必填 |
| 备注 | 文本域 | | 非必填 |
| 采购状态 | 下拉框 | 已申请 / 采购中 / 已完成 | 必填，默认「已申请」 |

#### 操作

- **新增**：资产管理员点击「新增采购申请」，弹出表单填写各字段
- **编辑**：资产管理员可修改已有记录
- **删除**：资产管理员可删除记录（建议软删除）
- **查看**：所有人均可查看全部数据（表格展示，支持分页、搜索、按列排序）
- **批量状态更新**：支持批量将选中记录标记为「采购中」或「已完成」

#### 预置数据

资产最小分类（初始值，后续可通过管理界面修改）：
- 办公设备
- 电子设备
- 网络设备
- 软件许可
- 实验设备
- 办公家具
- 其他

需求部门（初始值，后续可通过管理界面修改）：
- 研发部
- 测试部
- 产品部
- 运维部
- 行政部
- 财务部

### 2.2 权限管理

采用**无登录 + 管理员密码保护**模式：

- **普通访客**：直接访问系统 URL，可查看所有采购数据、透视报表、附件列表
- **资产管理员**：通过页面上的「管理员入口」输入预设密码后，解锁编辑功能（增删改）
- 密码存储在配置文件中，使用哈希值存储（非明文）
- 管理员登录状态通过 Session 保持，设置合理的过期时间（如 2 小时）

### 2.3 月度附件管理

#### 功能说明

- 按年月 + 部门维度管理附件
- 每个部门每个月可上传**多个**附件文件
- 主要用于归档各部门的审批邮件（通常为 .eml / .msg / .pdf 格式）

#### 操作

- 上传：选择年月和部门后，上传附件文件
- 查看/下载：所有人均可下载查看附件
- 删除：仅资产管理员可删除附件
- 列表展示：以年月为维度，展示各部门的上传情况（已上传数量）

#### 文件存储

- 文件存放目录：`./uploads/attachments/{年}/{月}/{部门}/`
- 数据库记录：文件名、原始文件名、上传时间、关联年月和部门
- 文件大小限制：单文件不超过 20MB

### 2.4 透视报表

#### 报表一：部门采购汇总

- 筛选条件：年份（必选）、月份（可选，不选则看全年）
- 展示内容：表格形式，每行一个部门，列包含：采购总金额、采购总数量、记录条数
- 支持按金额排序

#### 报表二：部门占比图表

- 筛选条件：同上
- 展示内容：饼图展示各部门采购金额占比
- 鼠标悬停显示具体金额和百分比

#### 报表三：资产分类分布

- 筛选条件：同上
- 展示内容：柱状图展示各资产分类的采购数量和金额

#### 报表四：导出 Excel

- 所有报表均支持导出为 Excel 文件
- 导出内容与页面展示一致

## 6. 数据导入方案

### 6.1 Excel 导入流程

1. 管理员在系统设置页面点击「导入历史数据」
2. 上传 Excel 文件（.xlsx 格式）
3. 系统解析 Excel，自动匹配列名与数据库字段
4. 展示预览：显示即将导入的数据条数和前 10 条预览
5. 管理员确认后执行导入
6. 返回导入结果：成功条数、失败条数、失败原因

### 6.2 列名映射规则

| Excel 列名 | 数据库字段 |
|-------------|-----------|
| 采购年份 | year |
| 采购月份 | month |
| 资产最小分类 | asset_category |
| 物品名称 | item_name |
| 生产厂家 | manufacturer |
| 物品型号 | model |
| 预算数量 | budget_qty |
| 预计单价 | unit_price |
| 总价 | total_price |
| 需求部门 | department |
| 需求人姓名 | requester_name |
| 需求人工号 | requester_id |
| 资产编码 | asset_code |
| 申请原因 | reason |
| 备注 | remark |
| 采购状态 | status |

### 6.3 容错处理

- 缺失的非必填字段：设为空值
- 缺失的必填字段：跳过该行并记录到错误日志
- 总价不一致时：以「数量 × 单价」重新计算覆盖
- 日期字段格式兼容：支持「2024年」「2024」等格式

## 8. 部署方案

### 8.1 内网部署

- 使用 Gunicorn 作为 WSGI 服务器
- Nginx 作为反向代理（可选，处理静态文件和 SSL）
- SQLite 数据库文件定期备份（可配置 cron 任务）

### 8.2 启动命令

```bash
# 安装依赖
pip install -r requirements.txt

# 初始化数据库
python init_db.py

# 生产环境启动（Gunicorn）
gunicorn -w 4 -b 0.0.0.0:5000 app:app

# 开发环境启动
python app.py
```

### 8.3 配置文件示例 (config.py)

```python
import os

class Config:
    SECRET_KEY = 'your-secret-key-change-in-production'
    SQLALCHEMY_DATABASE_URI = 'sqlite:///data/procurement.db'
    SQLALCHEMY_TRACK_MODIFICATIONS = False
    UPLOAD_FOLDER = os.path.join(os.path.dirname(__file__), 'uploads', 'attachments')
    MAX_CONTENT_LENGTH = 20 * 1024 * 1024  # 20MB
    ADMIN_PASSWORD_HASH = ''  # 通过 init_db.py 设置
    ADMIN_SESSION_HOURS = 2
```

## 10. 注意事项

1. **数据安全**：管理员密码使用 `werkzeug.security` 的 `generate_password_hash` / `check_password_hash` 处理，数据库文件需定期备份
2. **文件安全**：上传文件使用 UUID 重命名存储，防止路径穿越和文件名注入
3. **并发处理**：SQLite 在写入并发上有限制，Gunicorn workers 建议不超过 4 个，或考虑加文件锁
4. **浏览器兼容**：目标用户使用内网浏览器，建议支持 Chrome 90+ 和 Firefox 90+
5. **后续扩展**：如需对接公司域控、增加审批流程等，当前架构可平滑升级到 PostgreSQL + Flask-Login
